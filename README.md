# Database Design Project

This project demonstrates a relational database design using a fictional anime‑based dataset. It includes entity descriptions, data dictionaries, business rules, ER diagrams, relational schemas, and normalization steps. The work provides a structured example of how to design and document a database system from initial entities to final normalized tables.

## Key Components
- Entity descriptions and attributes
- Data dictionary with data types and constraints
- Business rules and relationship definitions
- ER diagram and relational schema
- Normalization (1NF, 2NF, 3NF)
- Sample SQL queries

## Skills Demonstrated
- Relational modeling
- ER diagramming
- Normalization
- SQL schema creation
- Query writing
- Documentation and data organization

## Files Included
- Databook PDF
- SQL schema file
- ER diagram components
